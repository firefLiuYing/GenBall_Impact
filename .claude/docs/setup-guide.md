# Setup Guide

This document provides instructions for setting up, building, and running the GenBall_Impact project.

## Requirements

- **Unity Version**: 2022.3.42f1c1
- **Platform**: Windows (primary), with potential cross-platform support
- **IDE**: Visual Studio, Rider, or VS Code

## Opening the Project

### Option 1: Open in Unity

1. Launch Unity Hub
2. Click "Add" and navigate to the project folder: `D:\Apps\Unity\Unity Project\GenBall_Impact`
3. Select Unity version 2022.3.42f1c1
4. Click on the project to open it

### Option 2: Open Solution in IDE

Open `GenBall_Impact.sln` in your IDE (Visual Studio, Rider, etc.)

**Note**: The solution file is generated by Unity. If it doesn't exist, open the project in Unity first, then go to Edit → Preferences → External Tools → Regenerate project files.

## Building

### Build through Unity Editor

1. Open Unity Editor
2. Go to **File → Build Settings**
3. Select target platform (Windows, Mac, Linux)
4. Click **Build** or **Build And Run**
5. Choose output directory

### Build Settings

- **Scenes in Build**: Ensure required scenes are added
  - `Launcher.unity` should be first (index 0)
  - Add `Prologue.unity`, `Episode1.unity` as needed

## Running

### In Unity Editor

1. Open the `Launcher.unity` scene
2. Press the **Play** button (or F5)
3. Game will start from the entry point

### Built Executable

1. Navigate to the build output directory
2. Run `GenBall_Impact.exe` (or equivalent for your platform)

## Scenes

Main scenes located in `Assets/Scenes/`:

### Production Scenes

- **`Launcher.unity`**: Entry scene - game initialization and startup
- **`Prologue.unity`**: Prologue chapter
- **`Episode1.unity`**: First episode

### Test/Development Scenes

- **`YueynScene.unity`**: Yueyn's test scene
- **`SSTest2.unity`**: SunnyStrike's test scene
- **`YueynTestMap.unity`**: Map system test scene
- **`SunnyStrikeScene.unity`**: SunnyStrike's development scene

## Development Workflow

### Typical Workflow

1. Open Unity Editor
2. Open a test scene (e.g., `YueynTestMap.unity`)
3. Make changes to scripts/prefabs
4. Press Play to test
5. Iterate

### Code Changes

- Scripts are located in `Assets/Scripts/`
- Changes to C# scripts require Unity to recompile
- Hot reload is supported for most changes

### Prefab Changes

- Prefabs are located throughout `Assets/`
- Changes to prefabs are saved automatically
- Use Prefab Mode (double-click prefab) for isolated editing

## Common Issues

### Missing References

If you see missing script references:
1. Check that all required packages are installed
2. Reimport the project: Assets → Reimport All

### Compilation Errors

If scripts won't compile:
1. Check the Console window for errors
2. Ensure Unity version matches (2022.3.42f1c1)
3. Try Assets → Reimport All

### Scene Won't Load

If scenes fail to load:
1. Check that the scene is added to Build Settings
2. Verify scene file exists in `Assets/Scenes/`
3. Check for missing dependencies
