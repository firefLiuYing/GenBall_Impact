# 经济系统与存档机制

## 设计原则

- 经济系统**非常简单**，目的仅是控制内容投放 + 提供更丰富的反馈
- **无反通胀设计**：游戏本身不依赖数值成长，不允许刷取，怪物掉落数据量不会随进程大幅提升

## 货币系统

### 圆形数据（Circular Data）

- **唯一获取方式**：在存档点存档时，根据枪中记录的击杀计数转换
- **转换率**：暂定 1:10（1击杀计数 → 10圆形数据）
- 后续转换效率可能获得提升

**设计目的**：
- 鼓励玩家主动击杀奥比斯（而非全部绕过去）
- 降低存档时击杀计数被重置的负反馈感

### 失落数据（Lost Data）

- **获取方式**：地图中探索获得（宝箱、隐藏区域等）
- 相对珍稀，可能以个数计
- 用于技能树节点的解锁

**设计目的**：
- 探索分支路径的多样化反馈
- 防止玩家通过无限刷奥比斯机关过快获得技能
- 控制技能树关键节点的进度

### 第三货币（待定）

- 可能用于更重要的强化或技能树关键节点
- 可能在关底/BOSS战掉落，或作为一章结尾隐藏道具全收集奖励
- **待策划确认是否需要**

## 道具与掉落

| 掉落物 | 获取方式 | 说明 |
|--------|---------|------|
| 击杀计数 | 怪物死亡时自动获取 | 不消耗，持续累积 |
| 生命回复道具 | 怪物掉落，需靠近拾取 | 回复半颗心，通过模块获取 |

## 存档机制（篝火式）

### 存档点（常规存档点）

等效于魂系游戏的篝火，在存档点可进行：
- 生命/护甲回复至满值
- 击杀计数转换为圆形数据，枪械进化阶段重置
- 技能树升级
- 枪械模块配置

### 存档记录内容

| 记录项 | 说明 |
|--------|------|
| 技能树状态 | 已解锁节点 |
| 货币数量 | 圆形数据 + 失落数据 + 可能的第三货币 |
| 角色属性等级 | 所有属性 |
| 怪物状态 | 每个怪物的死亡/存活 |
| 游戏进程 | 当前所在关卡位置 |
| 游戏时长 | |

### 怪物刷新规则

- **大部分怪物**：存档后不会刷新（永久死亡）
- **少数特定怪物**：存档后会刷新
  - 通常是：解开通道/触发机关所需的奥比斯
  - 决策依据：如果某个机关需要特定奥比斯才能通过，该奥比斯需要存档后可刷新

### 死亡与重载

- 主角死亡 → 读取上次存档的所有信息 → 重生在最后存档点
- 无论何时退出游戏，重新进入时也只读取上次存档状态

## 技能树框架

### 解锁方式

- 节点消耗：圆形数据、失落数据，或两者都需要
- 每种节点有自己的解锁条件

### 进度控制

- 第一章结束时期望推进到第三阶段节点
- 关键技能节点可能被失落数据卡住（因为无法刷取，依赖探索进度）
