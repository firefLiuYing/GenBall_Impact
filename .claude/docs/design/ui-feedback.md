# UI与反馈设计

## 剧情对话系统

### 基本形式

- 屏幕下方显示对话框
- 内容：**说话者名称 + 说话内容**
- 每条对话可编辑显示时长
- 事件触发方式：
  - 到达特定位置
  - 捡起特定物品
  - 击杀特定怪物
  - 演出中播放

### 第一章对话节点

- 苏醒后：小右初次登场（"喂，看看后边儿！"）
- 实验室门口：小右指导充能激活战斗功能
- 其他探索过程中的关键发现

## 战斗 HUD 反馈

### 设计原则

- 由于游戏中同屏奥比斯数量可能很多，**不做敌人血条**（信息密度过高、分散注意力、且信息算不上很有效）

### 准星反馈（核心 HUD 元素）

| 状态 | 反馈 |
|------|------|
| 散布 | 外框到中心的距离表示散布大小 |
| 命中 | 准星变色 / 十字线出现 |
| 击杀 | 特殊小形变或形状增强 |

### 枪械状态 HUD

- 生命/护甲（心形UI，6颗心）
- 弹匣余量
- 进化进度条 + 进化按钮提示（Q键）
- 能力枪械冷却状态
- 匣纳之枪：弹匣中奥比斯种类和顺序

## 视效反馈

### 射击相关

| 事件 | 视效 |
|------|------|
| 开火 | 枪口特效 |
| 子弹弹道 | 弹道拖尾 |
| 命中敌人 | 敌人受击动画 + 特效 |
| 击杀 | 额外增强特效 |

### 枪械进化演出

- 短时缓效果
- 枪械展开冲击波
- 枪械外观变化

## 音效反馈（规划阶段）

| 事件 | 音效 |
|------|------|
| 开火 | 各枪械开火音效 |
| 命中 | 命中反馈音效 |
| 击杀 | 击杀反馈音效 |

> 需要配合音频管理系统做较复杂调整，后续细化。

## 其他 UI

- **轮盘**：能力枪械切换选择
- **技能树界面**：在存档点打开
- **存档点交互**：按F交互后进入存档界面
- **怪物状态识别**：可能用 shader 给怪物做特殊描边表达状态
