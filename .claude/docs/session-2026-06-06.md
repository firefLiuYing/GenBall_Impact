# 2026-06-06 需求讨论结论

## 关键架构决策

1. **场景**：第一章单Scene + Prefab拖拽配置。Map/下为废弃代码。
2. **枪械进化**：根本性重设计——不再是单枪变形，而是"阶段=配装（主武器+配件）"，进化=切换到下一阶段配装。4阶段可配，只能前进不能回退，阶段上限通过全局事件动态解锁。
3. **Buff 瘦身**：回归临时属性修改器（可叠加+有时长）。模块/技能树/进化效果独立实现，不强制转为Buff。
4. **效果原语层**：Buff和配件共用效果原语（触发条件+效果），暂不做原语组合。新原语=策划提需求→程序写代码。
5. **配件系统**：独立 IModuleEffect，全局事件投放（GrantAccessory），不绑定到枪上。
6. **对话系统**：Excel编辑 + 导入下拉选ID。全部触发方式（位置/事件/互动/演出/闲聊）。三种暂停模式每条可配。三种结构（单条/序列/分支）。
7. **存档点UI**：World Space 3D UI（死亡搁浅风格全息面板），存档点（物理终端）与存档行为分离。
8. **触发器工具**：SceneEventTrigger + [SerializeReference] 多态参数，同一事件ID + 按类型动态参数。
9. **能力武器**：连理之枪/裁径之枪延后。先用匣纳之枪验证管线。
10. **第一章敌人**：只做4种（蓝/橙黄自爆/吞噬/酸液），飞行和相溶后续章节。
