# 策划需求核对清单

> **用途**：和策划开会逐条确认，避免需求模糊中断开发。
> **项目约束**：单人开发 + 单人策划，工具必须轻量实用。
> **最后更新**：2026-06-06

---

## 一、开发者已确定（开会时说明理由，策划无异议就定）

### 场景 & 关卡

| 决策 | 方案 | 理由 |
|------|------|------|
| 场景粒度 | 第一章一个 Unity Scene | 线性蚁穴结构，不需要场景切换逻辑 |
| 内容配置 | Unity Editor 拖 Prefab | 小团队最直观，Scene View 可视化 |
| 流程积木 | 全局事件 + 触发器/接收器模式 | 所有积木本质是"定义全局事件+实现不同触发器" |

### 对话系统

| 决策 | 方案 | 理由 |
|------|------|------|
| 内容编辑 | Excel 表格 | 策划友好，方便批量编辑和本地化 |
| ID 桥接 | Excel 导入后 Inspector 下拉列表选择 | 策划不用手动输入 ID |
| 触发方式 | 全部支持（位置/事件/互动/演出/闲聊） | — |
| 暂停模式 | 每条可配（完全暂停/可移动/完全自由） | 不同场景需要不同模式 |
| 对话结构 | 单条 + 序列 + 分支选项 | 架构统一支持 |
| 闲聊触发 | 定时随机 + 事件触发两种 | 枪"会说话"的设定 |

### 匣纳之枪

| 决策 | 方案 | 理由 |
|------|------|------|
| HUD 显示 | 弹匣图标 + LIFO 顺序 | 核心机制是栈顺序射出，玩家需要看到 |

### 存档系统

| 决策 | 方案 | 理由 |
|------|------|------|
| 存档点数量/位置 | 策划搭关卡时自己决定 | 与关卡设计紧密相关 |
| 交互 UI | World Space 3D UI（死亡搁浅风格全息面板） | 策划想要沉浸式场景内显示 |
| 存档 vs 存档点 | 分离设计：存档点是物理终端，存档行为也可在重要节点触发 | 灵活性 |

### Buff 系统

| 决策 | 方案 | 理由 |
|------|------|------|
| 职责范围 | 瘦身回归属性临时修改（可叠加/有持续时长） | 早期设计太上帝类，模块/技能树/进化效果应独立实现 |
| 配置方式 | Excel + 导入 | 同对话，策划友好 |

### Buff / 配件效果：原语层

| 决策 | 方案 | 理由 |
|------|------|------|
| 效果原语 | 抽象一层效果原语（触发条件+效果），Buff 和配件共用 | 避免写两份"伤害+25%" |
| 原语组合 | 暂不做 | 第一章只有"护甲满+50%伤害"需要组合，直接做一个原语更简单 |
| 新原语 | 策划提需求 → 程序写代码 | 二人团队，新原语频率低 |

### 模块/配件系统

| 决策 | 方案 | 理由 |
|------|------|------|
| 实现方式 | 独立于 Buff，使用效果原语 | 模块是永久装备（存档点配置），和 Buff（临时效果）生命周期不同 |
| 配件获取 | 通过全局事件投放，不绑定到枪上 | 获得枪的事件单独配置还送哪些配件 |

### 枪械进化系统

| 决策 | 方案 | 理由 |
|------|------|------|
| 核心模型 | 每个阶段=一套配装（1把主武器+若干配件），进化=切到下一阶段配装 | 不是一把枪变形，而是预设配装切换 |
| 阶段总数 | 4个，可配置（非硬编码） | 灵活性 |
| 每阶段负载上限 | 可配置（参考值 7/15/25/40，待策划确认） | — |
| 负载占用 | 只有配件占负载，武器不占 | — |
| 进化触发 | 击杀计数达标 → 提示按Q → 可选进化 | 玩家可以继续用当前阶段 |
| 进化方向 | 只能前进，不能回退 | — |
| 存档后 | 击杀清零 + 回阶段一，配装保留 | 下次玩不用重新配 |
| 阶段上限 | 动态解锁，通过全局事件控制 | 如首次到存档点→上限升到2；技能树节点→升到3 |
| 阶段上限持久化 | 存档保存当前上限 | — |
| 同武器重复 | 硬阻止，不同阶段不能选同一把武器 | — |
| 空阶段 | 强制配满，至少一把武器 | — |
| 默认配装 | 首次使用时自动填第一把可用武器+空配件 | 保证不会出现"没配装就进化"（配合阶段上限控制） |

### 技能树系统

| 决策 | 方案 | 理由 |
|------|------|------|
| 节点效果类型 | 全部支持：属性加成 / 解锁系统功能 / 解锁新能力 | 更像持久化能力提升，多种效果混合 |
| 节点结构 | 有前置依赖（必须先点A才能点B） | — |
| 节点数量 | 可配置，非硬编码 | 策划配多少就有多少 |
| 操作入口 | 存档点全息界面 | 同配装、存档一起 |
| 持久化 | 和配装一样保留，开新档才重置 | — |

### 经济系统

| 决策 | 方案 | 理由 |
|------|------|------|
| 击杀→货币转换率 | 可配置 | 策划随时调 |
| 失落数据投放量/位置 | 策划定 | 与关卡设计相关 |

### 触发器工具

| 决策 | 方案 | 理由 |
|------|------|------|
| 触发器类型 | 碰撞触发 / 手动交互 / 事件监听 | 各类型参数不同（碰撞需半径、交互需提示文本、监听需源事件ID） |
| 事件参数方案 | 同一事件ID + 按类型动态参数 | 如 GrantWeapon 带武器ID下拉框，而非 GrantWeapon_Pistol 拆事件 |
| 编辑器实现 | `[SerializeReference]` 多态 Params + Custom Editor | 选事件 → Inspector 自动切换对应参数子类，可行且工作量不大 |

### 能力枪械开发优先级

| 决策 | 方案 | 理由 |
|------|------|------|
| 连理之枪 / 裁径之枪 | 延后，先跑通第一章 | 用匣纳之枪验证能力武器管线即可 |
